Dependency Injection

Spring Dependency Injection (used with @Autowired annotation) helps you inject Spring component-objects in your app.
A Spring component is any class that Spring has been adviced 'watch' by giving it an annotation. Its a managed bean. Spring by defaults creates singletons when creating autowiring

Inversion of Control

In short, IoC is a design pattern that allows the calling class/container to provide their own dependency version. That is, if you DO NOT hardcode the instance of your dependency in your components.

Back end Communication

I used @Controller with creates a RESTful API that calls the backend for operations.

Bootstrap 3.7

Bootstrap is a CSS framework that makes it easy and convenient to make-up and style-up your application.
Developers get to focus on the meat of the project - building the business logic and the securities around it - not the cosmetics of the application. Since we started using Bootstrap, we stopped spending too much time in the mirrow :)

Spring Boot Web

JavaEE gives us an API to build web applications. Before SpringBoot-Web, we have to manually configure the JEE application - defining the web.xml for directives, and going through all the pain. Now, Spring-Web abstracts that away from the developer to help build efficient WebApps faster and nicer :)
Spring MVC is modeled also by this plugin. It helps resolve templates and manage business session objects (Controller --> Browser --> Controller)

Spring Boot Security

It would all be useless to build an application that would be in the public domain without securing it with the right weapons. Before The Spring guys built this tool, we used to write web-sessions and managed them manually. Upon successful login , we would create a token for a session, and attach it to the HTTP session, and on each back-end request, we would manually check in some hashtable if the session existed or was not expired.
Now Spring Security gives us a break from all the fault-prone setup. It takes away this setup routine, allowing for a quick application dev - once again!

Spring Boot JPA

See, Spring always makes sure to respect the rules of Java, and just as always, they follow the JRS documents when creating these cool tools. JPA allows us to manage data flow between the database and the business logic.
You'd normally find it in what is commonly known as the Data Layer which sits between the business logic and the database.
We used to write all SQL/MySQl commands and queries and flying them through the JDBC. So, some generous guys built a tool called Hibernate, which was taking away the pain of having to write SQL statements which, you know, most Java Devs would do bad at, and making sure that we focus on Java - what we do best.
Then Spring guys took Hibernate a step further - they wrapped Hibernate (which wraps-in JDBC by the way) within their tool (Spring JPA) aka Spring Data. Now Spring Data gives us basic CRUD operations without having to write a single line of code. Furthermore, if you have been a good-dev, following the JavaBeans standard, SpringData allows you to query your objects using CamelCasing in your interface repository method - making sure that you don't live another day to see SQL on your screen. You should be worried if you see yourself having to write queries within the SpringData framework (yes, they allow you to write your own SQL too - that is to give backward compatibility for stinking-code hahaha)

Spring Thymeleaf

Thymeleaf is just an enhanced version JSP. Just not as beautiful as Angular - just easier to use compared to JSP and JSF. But it actually gives you a front-end display for your business data. This is where we bind business fields and object. It is really just a managed HTML
